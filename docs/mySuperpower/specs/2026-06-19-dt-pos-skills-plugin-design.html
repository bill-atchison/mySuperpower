Overview

dt-pos-skills is a new, standalone coding-agent plugin that bundles the Dollar Tree Point-of-Sale documentation toolkit so it can be installed independently of the mySuperpower fork and shared with the DT team across multiple harnesses. It packages four skills, six convenience slash commands, and a slim bootstrap, with per-harness manifests for Claude Code, Codex, and Gemini CLI.

These skills currently live (uncommitted) inside mySuperpower/skills/. Leaving them there would leak them into the mySuperpower plugin build and tangle DT-specific tooling with a general-purpose fork. This plugin gives them a clean, purpose-built home.

Goals

  • One installable plugin that delivers the DT feature-docs, release-info, and release-QA skills plus their / commands.
  • Self-contained: works even when no other Superpowers plugin is present.
  • Multi-harness from day one (Claude Code, Codex, Gemini CLI).
  • No build pipeline — skills committed in final form.

Non-goals (YAGNI)

  • No pristine-source mirror / build-overlay / release branch (mySuperpower needs those to absorb upstream merges; this plugin has no upstream).
  • No Cursor / Kimi / OpenCode / Pi support at launch.
  • No public/open-source distribution — internal DT tooling only.

Chosen Approach

Approach A  No-build, distribution-only multi-harness plugin. Because there is no upstream to track and no rebrand to apply, skills are authored directly with their final dt-pos-skills: namespace and committed as-is. The plugin ships per-harness manifests, a marketplace catalog, and its own slim session-start bootstrap, served straight from the GitLab repo root.

Rejected: B — mirror mySuperpower's pristine-source + build-overlay + release-branch machinery (pure ceremony with no upstream); C — Claude-Code-only (the team chose multi-harness).

Why no build

mySuperpower's build exists to keep a byte-pristine upstream mirror and apply HTML/namespace overlays at build time. dt-pos-skills authors everything in final form, so the filesystem is the plugin. The marketplace is served from the repo root (source: "./") — no dist/, no release branch.

Data Flow & Cross-References

Namespace rewrites (once, at authoring)

Every my-superpower:creating-feature-docs / release-info / release-qa-tests reference — in the 6 commands and the skills' internal cross-links — is rewritten to dt-pos-skills:.

External-dependency softening

release-qa-tests references my-superpower:dispatching-parallel-agents, which is core Superpowers and is NOT bundled. It is softened to harness-neutral guidance ("dispatch parallel agents, ~4 tickets per batch") with an optional "use your platform's parallel-agents skill if available" note, so standalone users are not broken.

Branding

All five templates already inline the DT brand.css (palette, Calibri, off-white) and render offline from file://. The bundled dollar-tree-branding skill is the canonical source the skills cite; copied from team-claude/plugins/dollar-tree-branding (reconciled against ~/.claude/skills/dollar-tree-branding at copy time).

Testing & Verification

  • Manifest validity: each plugin.json / marketplace.json / gemini-extension.json parses as JSON; bump-version.sh --check reports all in sync.
  • Namespace sweep: no residual my-superpower: / superpowers: in skills/ or commands/ beyond the softened note.
  • Self-containment: the 5 templates render offline from file:// with correct DT branding.
  • Acceptance test (Claude Code): fresh session → /feature-docs <ticket> resolves the dt-pos-skills:creating-feature-docs skill and follows it; /release-qa-tests and /release-info resolve their skills.
  • Source cleanup: mySuperpower working tree clean; its build no longer emits the 3 DT skills.
